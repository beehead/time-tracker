# Time Tracker

Программа для учёта временных активностей с поддержкой категорий и экспорта данных.

## Функциональность

- Учёт активностей по категориям:
  - Продуктивная активность
  - Инвестиционная активность
  - Лишняя активность
  - Активный отдых
  - Дом и быт
  - Минимальное техобслуживание
- Ведение временных слотов (time slots)
- Хранение данных в SQLite
- Возможность экспорта
- Ввод через Telegram-бота
- **Добавление уже прошедших событий**

## Структура проекта

```bash
src/
├── models/
```

## Зависимости

- Python 3.7+

## Запуск

1. Установите зависимости:

   ```bash
   pip install -r requirements.txt
   ```

2. Создайте файл `.env` на основе шаблона:

   ```bash
   cp .env.example .env
   ```

3. Получите токен у @BotFather в Telegram и добавьте в `.env`:

   ```bash
   # Отредактируйте .env и добавьте:
   TELEGRAM_BOT_TOKEN=ваш_токен_от_BotFather
   ```

   > 📝 **Как получить токен:**
   > 1. Напишите @BotFather в Telegram
   > 2. Выполните команду /newbot
   > 3. Введите имя бота и username
   > 4. Скопируйте полученный токен в .env файл
   >
   > ⚠️ **Важно:** Никогда не коммитьте `.env` файл! Подробнее см. [docs/SECRETS.md](docs/SECRETS.md)

4. Запустите приложение:

   ```bash
   python -m src.main
   ```

   Бот будет работать в режиме **polling** — периодически проверяет Telegram API на новые сообщения.
   Это работает везде: на локальном компьютере, на VPS, без требований к публичному IP.

**Остановка:** Нажмите `Ctrl+C` в терминале.

## Запуск в Docker

Для запуска в контейнере используйте Docker Compose:

```bash
# 1. Создайте .env на основе шаблона
cp .env.docker.example .env

# 2. Добавьте TELEGRAM_BOT_TOKEN в .env
# TELEGRAM_BOT_TOKEN=ваш_токен

# 3. Запустите контейнер
docker-compose up -d

# 4. Проверьте логи
docker-compose logs -f time-tracker-bot
```

Подробнее: [docs/DOCKER.md](docs/DOCKER.md)

## Команды бота

- `/start` — приветствие и справка по командам
- `/start_activity <название> [тип]` — начать отслеживание активности
  - Типы: `продуктивная`, `инвестиционная`, `лишняя`, `отдых`, `дом`, `техобслуживание`
- `/stop_activity` — остановить текущую активность
- `/status` — показать текущую активность
- `/add_past <название> [тип]` — добавить уже прошедшее событие
  - После команды бо�� попросит указать:
    1. Время начала события (формат: HH:MM или HH:MM:SS)
    2. Длительность события (форматы: 30m, 1h, 1h 30m, 90 минут)
- `/export` — экспортировать все данные в текстовом виде
- `/cancel` — отменить текущую операцию

## Примеры использования

### Начать активность
```
/start_activity работа продуктивная
```

### Остановить активность
```
/stop_activity
```

### Добавить прошедшее событие
```
/add_past спорт отдых
```
Затем указать время начала и длительность.

### Экспортировать данные
```
/export
```

## Хранение данных

Данные сохраняются в `data/time_tracker.db` (SQLite).
Папка `data/` создаётся автоматически.
